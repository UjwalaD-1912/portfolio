# Free Deployment Guide for Salesforce Portfolio

This guide will help you deploy your Salesforce portfolio website for **FREE** using various hosting platforms.

## ðŸš€ Free Hosting Options

### 1. GitHub Pages (Recommended)
**Cost:** Completely Free
**Requirements:** GitHub account (free)

#### Steps:
1. **Create GitHub Account**
   - Go to [github.com](https://github.com)
   - Sign up for free account

2. **Create New Repository**
   - Click "New Repository"
   - Name it: `your-username.github.io` (replace with your GitHub username)
   - Make it Public
   - Check "Add a README file"

3. **Upload Your Files**
   - Click "uploading an existing file"
   - Drag and drop all files: `index.html`, `styles.css`, `script.js`, `README.md`
   - Commit changes

4. **Enable GitHub Pages**
   - Go to repository Settings
   - Scroll to "Pages" section
   - Source: "Deploy from a branch"
   - Branch: "main"
   - Save

5. **Access Your Website**
   - Your site will be live at: `https://your-username.github.io`
   - Takes 5-10 minutes to deploy

### 2. Netlify
**Cost:** Free (with Netlify branding)
**Features:** Custom domains, form handling, continuous deployment

#### Steps:
1. Go to [netlify.com](https://netlify.com)
2. Sign up for free
3. Drag and drop your project folder to Netlify dashboard
4. Get instant deployment with random URL
5. Optional: Configure custom domain

### 3. Vercel
**Cost:** Free tier available
**Features:** Fast deployment, custom domains, analytics

#### Steps:
1. Go to [vercel.com](https://vercel.com)
2. Sign up with GitHub account
3. Import your GitHub repository
4. Automatic deployment on every commit

### 4. Surge.sh
**Cost:** Completely Free
**Features:** Simple command-line deployment

#### Steps:
1. Install Node.js from [nodejs.org](https://nodejs.org)
2. Open terminal/command prompt
3. Run: `npm install -g surge`
4. Navigate to your project folder
5. Run: `surge`
6. Follow prompts to get free domain
